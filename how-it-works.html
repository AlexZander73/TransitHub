How CoastPulse Transit Atlas Works

The frontend is fully static and deploys directly to GitHub Pages. Core datasets are JSON files committed in the repository, then loaded client-side by modular vanilla JavaScript services.

Runtime data modes

  1. Live mode (optional): if enabled and fresh, departures.live.json and alerts.live.json are used.
  2. Sample snapshot mode: if live data is missing or stale, sample JSON is used.
  3. Scheduled fallback: departures can be generated from route frequency profiles when snapshots are absent.

Source state is surfaced in the UI so fallback conditions are transparent.

Direct-estimate boundaries

PlannerLite only supports direct same-route journeys (explicit edge or same route sequence). Transfer-capable locations are labeled, but full transfer journey planning is intentionally out of scope in this static deployment.

Expansion architecture

Region-aware datasets (regions/stops/routes/lines/interchanges/patterns) allow Brisbane and broader SEQ areas to be added without rewriting core UI architecture.
