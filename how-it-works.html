How CoastPulse Transit Works

The site is a static web app. Core route, stop, and map data live in JSON files committed to the repository and served directly via GitHub Pages.

Data Modes

  1. Live mode (optional): if configured and fresh live JSON exists, departures/alerts are shown from live files.
  2. Sample snapshot mode: if no live file is available, sample snapshot departures are used.
  3. Scheduled fallback: if needed, departures are generated from route frequency profiles.

The UI always surfaces fallback status so users can see when live data is unavailable.

Direct Travel Estimates

Only direct route/line estimates are supported. If origin and destination are not directly connected on a supported route pattern, the app clearly states that transfer journey estimates are not available yet.

Expansion Path

The data model is region-ready. Brisbane and other SEQ areas can be introduced by adding region-tagged datasets and map layers, without replacing frontend architecture.
